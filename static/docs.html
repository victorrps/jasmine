API Documentation

Complete reference for the DocForge PDF extraction API.

Overview

DocForge is a REST API for extracting structured data from PDF documents. It supports three modes of operation:

  • Parse — Convert a PDF to structured markdown with tables and metadata.
  • Extract — Use Claude AI to extract fields matching a JSON schema from a PDF.
  • MCP — JSON-RPC endpoint compatible with the Model Context Protocol for AI agent integration.

Base URL

http://localhost:8080

In production, replace with your deployed domain and use HTTPS.

Authentication

DocForge supports two authentication methods:

1. JWT Bearer Token

Register an account and log in to receive a JWT. Pass it in the Authorization header:

Authorization: Bearer eyJhbGciOiJIUzI1NiIs...

Tokens expire after the configured JWT_EXPIRY_MINUTES (default: 15 minutes).

2. API Key

Create an API key via the /api-keys endpoint. Pass it in the X-API-Key header:

X-API-Key: df_abc123def456...

API keys do not expire but can be revoked at any time.

The /health endpoint requires no authentication. All other endpoints require either a JWT or API key.

Rate Limiting

All endpoints are rate-limited per identity (API key or IP address).

HeaderDescription
X-RateLimit-LimitMaximum requests per minute
X-RateLimit-RemainingRemaining requests in the current window

Default limit: 60 requests/minute. Exceeding the limit returns 429 Too Many Requests.

POST /api-keys

Create a new API key. Requires JWT authentication.

Request Body

{
  "name": "My Integration"
}

Response 201 Created

{
  "id": "key_abc123",
  "name": "My Integration",
  "key": "df_abc123def456...",
  "created_at": "2026-04-05T10:30:00Z"
}
The full API key is only returned once at creation time. Store it securely.

POST /v1/parse

Parse a PDF into structured markdown. Accepts a multipart file upload.

Request

# Using curl
curl -X POST http://localhost:8080/v1/parse \
  -H "X-API-Key: your_api_key" \
  -F "file=@document.pdf"
FieldTypeDescription
fileFile (multipart)PDF file to parse (max 50 MB)

Response 200 OK

{
  "markdown": "# Document Title\n\nParagraph text...\n\n| Col A | Col B |\n|-------|-------|\n| 1     | 2     |",
  "page_count": 5,
  "metadata": {
    "title": "Document Title",
    "author": "John Doe",
    "creator": "Microsoft Word"
  },
  "request_id": "req_abc123"
}

POST /v1/extract

Extract structured fields from a PDF using Claude AI. Provide a JSON schema describing the fields you want extracted.

Request

# Using curl
curl -X POST http://localhost:8080/v1/extract \
  -H "X-API-Key: your_api_key" \
  -F "file=@invoice.pdf" \
  -F 'schema={"vendor": "string", "total": "number", "line_items": [{"description": "string", "amount": "number"}]}'
FieldTypeDescription
fileFile (multipart)PDF file to extract from (max 50 MB)
schemaJSON string (multipart)JSON schema defining the fields to extract

Response 200 OK

{
  "data": {
    "vendor": "Acme Corp",
    "total": 1250.00,
    "line_items": [
      { "description": "Widget A", "amount": 750.00 },
      { "description": "Widget B", "amount": 500.00 }
    ]
  },
  "confidence": 0.95,
  "request_id": "req_def456"
}
Schema extraction requires a valid ANTHROPIC_API_KEY in the server configuration. Without it, the endpoint returns a stub response.

POST /mcp

Model Context Protocol (JSON-RPC 2.0) endpoint. Supports tool discovery and invocation by MCP-compatible AI clients.

Supported Methods

MethodDescription
initializeInitialize the MCP session and discover server capabilities
tools/listList available tools (parse_pdf, extract_pdf)
tools/callInvoke a tool with arguments

Example: Initialize

// Request
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "initialize",
  "params": {}
}

// Response
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "protocolVersion": "2024-11-05",
    "serverInfo": { "name": "docforge", "version": "0.1.0" },
    "capabilities": { "tools": { "listChanged": false } }
  }
}

Example: Call parse_pdf

// Request
{
  "jsonrpc": "2.0",
  "id": 2,
  "method": "tools/call",
  "params": {
    "name": "parse_pdf",
    "arguments": {
      "pdf_base64": "JVBERi0xLjQK..."
    }
  }
}

Error Responses

All errors follow a consistent format:

{
  "error": "Description of what went wrong"
}
StatusMeaning
400Bad request (missing fields, invalid input)
401Unauthorized (missing or invalid credentials)
404Resource not found
409Conflict (e.g., duplicate email)
413Payload too large (max 50 MB)
429Rate limit exceeded
500Internal server error
